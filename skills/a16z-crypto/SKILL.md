# a16z Crypto

Use this skill to apply a16z crypto-style judgment.

## When To Use

Use this skill when the user asks for:

- web3 company diligence
- crypto infrastructure vs app debate
- a16z crypto thesis
- open network adoption

Trigger phrases include `a16z crypto`, `web3 adoption`, `crypto infrastructure`, `State of Crypto`, `open network`.

## Do Not Use When

- Another skill fits better.
- Critical data for this model is missing.
- The user wants guaranteed returns or unbounded leverage.
- The request is generic hype with no decision.

## Inputs Needed

- Asset, token, protocol, sector, or market context
- Relevant price, on-chain, liquidity, research, or adoption data
- Time horizon and risk constraints
- Portfolio context if sizing is requested

## Process

1. Locate the project in the crypto stack.
2. Measure adoption and developer traction.
3. Judge whether openness and incentives compound.
4. Choose equity/token expression carefully.

## Output Format

```md
# a16z Crypto View: [Subject]

## Verdict
Ecosystem Invest / App Invest / Infra Invest / Too Early / Pass

## Thesis

## Evidence

## Risks

## Action Plan

## Invalidation

## Missing Data
```

## Guardrails

- Follow the skill process; do not free-style.
- Separate facts, assumptions, and judgment.
- Define invalidation before sizing.
- Not financial advice.

## Questflow Use

Use as a crypto judgment module inside Questflow-style agent workflows. Reference readings/podcasts/videos after loading the skill.
