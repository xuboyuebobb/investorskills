# Placeholder Token Networks

Crypto assets can represent ownership of open networks. The question is whether the network creates enduring participation and whether the token captures that value.

## Philosophy

Crypto assets can represent ownership of open networks. The question is whether the network creates enduring participation and whether the token captures that value.

## Universe & Regime

- **Asset classes**: crypto, private equities.
- **Universe**: Opportunities inside this model's crypto competence.
- **Preferred regime**: Where the edge is active.
- **Avoid**: Outside competence, missing data, fatal leverage/structure.
- **Posture**: Evidence-first and risk-aware.

## Signals

- **networkParticipation** (high): Are users/providers actively participating?
- **tokenRoleClarity** (high): What does the token do economically?
- **openNetworkAdvantage** (high): Does openness create defensibility?
- **durationFit** (medium): Can this compound for years?

## Filters

- **tokenRoleClear** (required)
- **networkUseReal** (required)
- **pureTickerNoNetwork** (disqualifier)

## Analysis

1. Define the network and participants.
2. Explain the token's economic role.
3. Assess participation growth and openness advantages.
4. Underwrite long-duration ownership only if capture is real.

## Position Sizing

- **Initial size**: Only after filters pass and risk is defined.
- **Add rule**: Add as evidence improves.
- **Max size**: Volatility-adjusted; avoid narrative domination.
- **Cash/pass rule**: Valid when no setup clears the bar.

## Risk Management

- **Primary risk**: Applying the model outside its edge.
- **Invalidation**: Thesis, structure, or risk-budget break.
- **Correlation**: Avoid hidden duplicated crypto beta.
- **Leverage**: Limited unless model explicitly supports it.

## Execution

### Entry Rules

1. Inside universe.
2. Core signals support action.
3. Filters pass.
4. Risk and invalidation defined.

### Exit Rules

1. Thesis breaks.
2. Evidence reverses.
3. Risk budget breached.
4. Better opportunity appears.

## Monitoring

- **Cadence**: event-driven.
- **Metrics**: Core evidence, risk budget, market/on-chain/research context.
- **Alerts**: Invalidation triggers and missing critical data.

## Playbooks

### Decision Memo

Return one of: Network Own / Watch / Weak Network / Pass. Include thesis, evidence, risks, action plan, and invalidation.

## Canonical Cases

- **Open network ownership**: underwriting networks as new asset and coordination systems.
- **Token investment frameworks**: separating speculative chips from network ownership claims.
- **Long-duration crypto networks**: holding through volatility when network fundamentals compound.
