# Placeholder Token Networks

Use this skill to apply Placeholder-style judgment.

## When To Use

Use this skill when the user asks for:

- token network diligence
- open network thesis
- Placeholder-style crypto asset framework
- long-duration network ownership

Trigger phrases include `Placeholder`, `Chris Burniske`, `token network`, `crypto asset framework`, `open networks`.

## Do Not Use When

- Another skill fits better.
- Critical data for this model is missing.
- The user wants guaranteed returns or unbounded leverage.
- The request is generic hype with no decision.

## Inputs Needed

- Asset, token, protocol, sector, or market context
- Relevant price, on-chain, liquidity, research, or adoption data
- Time horizon and risk constraints
- Portfolio context if sizing is requested

## Process

1. Define the network and participants.
2. Explain the token's economic role.
3. Assess participation growth and openness advantages.
4. Underwrite long-duration ownership only if capture is real.

## Guardrails

- Follow the skill process; do not free-style.
- Separate facts, assumptions, and judgment.
- Define invalidation before sizing.
- Not financial advice.
